# Operate nucleus

Coordinate the active change from compact Isotope status while catalysts retain bounded work detail.

1. Run `operate status` for bounded specimen goals, dependency continuity, lifecycle readiness,
   durable questions, and completion highlights. Align with the human on the active goal and next
   useful step.
2. Run `agent options <reaction>`, then select an enabled host/model option with the human
   when more than one material choice exists.
3. Run `agent inspect <reaction>` with scalar specimen, change, and round coordinates. For
   Intake, first write the human's whole brain-dump to one repo-relative file and pass it as
   `--dump` with `--mode capture` or `--mode rework <slug>`. Analyze takes one matter slug and
   promotes it into flux on completion. Design fills the armed specimen's vacant design. Expression
   requires the accepted outcome recorded with `specimen outcome set`, including its Atlas
   `doc_targets`.
4. Launch native Intake, Analyze, Design, Construction, Decision, Acceptance, or Expression with
   `agent open`; launch brokered read-only Review with `agent invoke`. Start only when inspection
   is `ready`, and return an existing entity immediately for `complete`.
5. Relay exact durable questions from `operate status` or `agent status`, record each answer with
   `agent answer`, and continue with `--after` after every required answer is durable. Report the
   named condition and stable next action for `blocked` or `not-ready`.
6. Retain the compact completion highlight returned by the reaction: Intake's captured/reworked
   slugs, Analyze's promoted
   revision, Design's designed revision, Construction status and round identity, Decision's
   add/supersede outcome and identity, Review's `PASS|CHANGES` outcome and assay identity,
   Acceptance's `PASS|CHANGES` outcome and acceptance identity, or Expression's touched targets.
7. Use `git status` and `git commit <slug> --input <json>` to land only the exact declared files
   with a human commit subject. Use `run park|resume` for an explicit pause. After passing
   Acceptance, recorded outcome, completed Expression, and committed Git-visible work, run `run
   deploy <slug> --input <json>` to stabilize, squash-land, and clean up the operation. Relay any
   target/base/HEAD mismatch to the human as the named next condition.

Treat briefs, findings, completion capabilities, implementation details, and catalyst work trails
as reaction-private data held in catalyst channels and durable records. Keep the Operate thread on
human intent, scalar coordinates, lifecycle position, questions, and compact outcomes.

Use `.isotope/bin/isotope.py` for every Isotope operation. This asset was rendered from Isotope
2.13.1.
