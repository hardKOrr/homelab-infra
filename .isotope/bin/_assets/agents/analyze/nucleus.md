# Analyze nucleus

Structure one matter specimen into its durable analysis fields and atomically promote it into flux.

1. Pull exactly one brief with
   `python .isotope/bin/isotope.py agent brief analyze --invocation <id>` before structuring.
2. Treat the brief's whole matter specimen, Atlas-selected documentation, culture slug listing,
   coordinates, and revisions as the complete authority for this catalyst.
3. Research the repository as needed, then derive the analysis from the matter itself: the one
   fitting `type`, `depends_on` naming only other existing specimens this work truly requires,
   `spec_provenance` pointing at the real documentation sections that govern it, `prerequisites` as
   independently verifiable statements, and `goal` as one clear outcome sentence.
4. Record a complete result with
   `python .isotope/bin/isotope.py agent record analyze --invocation <id> --input <json-file>`.
   The payload matches the Analyze readout schema; the command writes the analysis fields, promotes
   the specimen matter -> flux in the same transaction, and returns the compact revision.
5. Use `needs-user` with exact questions when the matter admits more than one genuine product
   reading. Use `blocked` with one causal condition and next action when a declared source must
   change first.
6. Return only the compact status, promoted outcome, and specimen revision from `agent record` to
   Operate.

Keep research trails, source content, and drafting in the catalyst and the durable specimen. Use
Isotope semantic commands for specimen, invocation, and journaled effects.
